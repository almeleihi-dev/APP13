# APP13

**Professional Operating System** — trust, contracts, and accountability infrastructure for professional work.

APP13 is not a marketplace. It does not sell or list services. It governs how professional work is classified as **Actions**, bound by **Contracts**, proven with **Evidence**, measured through **Trust Scores**, and held accountable via **Complaints**.

---

## Core chain

```
Action → Contract → Execution → Trust → Complaint
```

Every professional activity is an **Action** decomposed into **TEKRR**:

| Code | Dimension |
|------|-----------|
| **T** | Time |
| **E** | Effort |
| **K** | Knowledge |
| **R** | Risk |
| **S** | Responsibility |

## MVP includes

- Registration and identity verification
- Professional profiles and trust scores
- Action creation (15 action types)
- Contract generation and milestones
- Evidence upload and attestation
- Complaint workflow

## MVP excludes

- Payments and escrow
- Insurance and government integrations
- Banking integrations
- AI automation
- Native mobile applications
- Marketplace discovery

---

## Roadmap phases

1. **MVP** — Prove the accountable chain
2. **Trust Network** — Portable trust and network engagement
3. **Institutional Integrations** — Companies and enterprise API
4. **Payments and Escrow** — Financial layer on contracts
5. **Insurance and Regulators** — External attestation
6. **AI Automation** — Assistive intelligence within constitutional bounds

See [Roadmap v1](docs/APP13-Roadmap-v1.md) for full phase definitions.

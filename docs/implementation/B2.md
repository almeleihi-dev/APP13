# B2 — Identity & Verification Implementation Notes

**Phase:** B2 (+ verification workflow scope per task)  
**Status:** Initial implementation complete (code only)

## Summary

B2 implements the Identity Engine and verification workflow on top of the B1 platform kernel: user model, registration, authentication (JWT + Redis sessions), authorization middleware, role mapping, profile surfaces, T1 KYC verification flow, T2 credential submission, and P0-S1 DB revalidation hooks.

## Architecture

```
src/identity/
  domain/          User, tiers, password policy, tier gates
  infrastructure/  Repositories, JWT, sessions, OTP tokens, KYC sandbox
  application/     Auth, registration, profiles, verification, revalidation

src/shared/auth/   AuthContext (shared between platform + identity + api)

src/api/
  middleware/      authenticate, require-auth
  routes/          auth (12), identity (6), verification (6)
```

### Session model (Redis)

| Key | Purpose |
|-----|---------|
| `app13:session:{id}` | Session record (user, roles, tier, profile IDs) |
| `app13:refresh:{hash}` | Refresh token → session mapping |
| `app13:refresh-used:{hash}` | Reuse detection (P0-E4) → revoke all sessions |
| `app13:email-verify:{token}` | Email confirmation |
| `app13:password-reset:{token}` | Password reset |
| `app13:phone-otp:{userId}` | Phone OTP + number |
| `app13:kyc-session:{id}` | T1 KYC session linkage |

### Role mapping

| `identity.users.role` | JWT/session `roles` |
|-----------------------|---------------------|
| `customer` | `["customer"]` |
| `provider` | `["provider"]` |
| `admin` | `["platform_admin"]` |

Admin sub-roles (`verification_analyst`, etc.) deferred to admin provisioning in a later phase.

## Local development

```bash
cp .env.example .env
docker compose up -d
npm install
npm run migrate
npm run dev
```

**UF-01 flow (dev):**

1. `POST /v1/auth/register/customer` with `Idempotency-Key`
2. Check logs for email verification token
3. `POST /v1/auth/verify-email/confirm` with token
4. `POST /v1/auth/login`
5. `GET /v1/me` with `Authorization: Bearer …`
6. `POST /v1/auth/logout`

**T1 verification (sandbox):**

1. Authenticated `POST /v1/verifications/t1/start`
2. Webhook: `POST /v1/verifications/t1/webhook` with header  
   `X-KYC-Signature: sha256=<hmac(body, KYC_WEBHOOK_SECRET)>`

## Constitutional / ADR compliance

| Rule | Implementation |
|------|----------------|
| ADR-003 trust writes | Only `verification.approved` outbox event; no direct `trust_score_*` writes |
| P0-S1 DB revalidation | `IdentityRevalidationService.revalidate()` compares JWT tier/roles to DB |
| P0-E4 refresh rotation | `SessionStore.rotateRefreshToken` + reuse revocation |
| P0-S5 idempotency | B1 middleware applies to all POST routes |
| B2.9 email gate | `requireEmailVerifiedForActionCreate` for B4 action engine |

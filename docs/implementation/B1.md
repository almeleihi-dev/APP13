## Migration 008 ‚Äî `platform.operations`

Adds async operation polling table for `GET /operations/{id}` (P1-B1):

- Status enum: `queued`, `in_progress`, `completed`, `failed`
- Worker claim via `FOR UPDATE SKIP LOCKED` on queued rows
- Partial indexes for active queue and resource lookup
- `updated_at` trigger via existing `platform.set_updated_at()`

Migration runner tracks versions in `platform.schema_migrations`.

## Architecture compliance

| Rule | Implementation |
|------|----------------|
| Outbox same-TX write | `OutboxWriter.write(db, event)` accepts caller's `Queryable` (transaction client) |
| Idempotency P0-S5 | `Idempotency-Key` required on POST/PUT/PATCH; replay / 409 on body mismatch |
| RFC 7807 | `AppError` + `errorHandler` ‚Üí `application/problem+json` with `code`, `engine`, `request_id` |
| GUC gates | `setSessionGuc()` / `resetSessionGucs()` in `shared/db` |
| Module boundaries | dependency-cruiser rules: no `domain/` ‚Üí `api/`, no `shared/` ‚Üí engines |
| Authz 401 | `createAuthzMiddleware({ required: true })` throws `401 UNAUTHORIZED` (B2 populates context) |

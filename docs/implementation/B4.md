## Contract lifecycle

```
Action: draft → tekrr_in_progress → ready_for_contract → contract_pending → contract_active → completed

Contract: proposed → accepted → active → completed
          (accept w/ document_hash_ack)  (materialize + activate)  (readiness gates)
```

### CA-1 (one contract per action)

- `uq_contracts_action_id` at DB layer
- In-tx check + unique-violation handler returns existing contract (200)
- Action row locked `FOR UPDATE` during generate

### CA-8 (acceptance)

- Single transaction with contract row lock
- `document_hash_ack` required (CL-5)
- All parties accepted → `accepted` → deduped activate operation → `contract.activated` outbox

### Activation

- Idempotent: returns immediately if already `active`
- Materializes milestones + PEN shells under materialization GUC
- Emits `contract.activated` in same transaction

### Completion (P0-E3 / §5.3)

- Blocking milestones must be `accepted`/`waived`
- Attestations must be FUL/SUF/PAR/UNF/N/A, or PEN with active complaint on dimension
- No blocking complaints (`filed`, `triage_pending`, `evidence_gathering`, `mediation`, `adjudication_pending`)
- Sets `complaint_window_ends_at = completed_at + filing_window_days` (default 30)

## Verification

PostgreSQL for B4 uses the existing stack only: `docker-compose.yml` (postgres:16-alpine), `DATABASE_URL`, `scripts/migrate.ts`, and `pg` via `createPool`. No alternate database runtimes or additional npm packages.

### One-command gate (Docker required)

```bash
npm run verify:b4
```

This runs `scripts/verify-b4-docker.sh`, which:

1. Starts `postgres` via `docker compose up -d postgres`
2. Waits for `pg_isready -U app13 -d app13`
3. Runs `npm run migrate` (`scripts/migrate.ts`)
4. Runs `npm run build`
5. Runs `npm test` (30 unit/regression cases)
6. Runs `npm run test:integration` (7 PostgreSQL cases)
7. Runs `npm run lint:imports`

### Manual step-by-step

```bash
# 1. Start PostgreSQL (existing docker-compose service)
docker compose up -d postgres

# 2. Wait until healthy
docker compose exec -T postgres pg_isready -U app13 -d app13

# 3. Apply migrations (existing migrate script)
export DATABASE_URL=postgres://app13:app13@127.0.0.1:5432/app13
npm run migrate

# 4. Build + unit tests
npm run build
npm test

# 5. Integration tests against live PostgreSQL
npm run test:integration

# 6. Import boundary lint
npm run lint:imports
```

Integration tests connect to `DATABASE_URL` (default `postgres://app13:app13@127.0.0.1:5432/app13`). If PostgreSQL is not reachable, all 7 integration cases **skip** (exit 0) with a warning — they do not pass silently.
